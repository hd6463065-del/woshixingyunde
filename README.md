対象テーブルの必須チェック
2.1 補正対象テーブルが選択されていない場合
「対象テーブルを選択してください」のエラーメッセージを表示する
当イベントの処理を終了する
2.2 対象テーブルが選択されている場合
次のステップ（Excel ファイルの読み込み）に進む
3. Excel ファイルの読み込み
3.1 選択された Excel ファイルをデータテーブル形式で読み込む
3.2 読み込み中にエラーが発生した場合
「ファイルの読み込みに失敗しました」のエラーメッセージを表示する
エラー表示領域を表示する
当イベントの処理を終了する
4. 処理区分列の存在チェック
4.1 Excel ファイル内に「処理区分」列が存在しない場合
「処理区分列が存在しません」のエラーメッセージを表示する
エラー情報を保存する
当イベントの処理を終了する
4.2 「処理区分」列が存在する場合
次のステップ（ファイル項目数のチェック）に進む
5. ファイル項目数のチェック
5.1 選択された対象テーブルの定義に基づき、Excel ファイルの列数が一致するかチェックする
5.2 列数が一致しない場合
「ファイルの項目数が正しくありません」のエラーメッセージを表示する
エラー情報を保存する
当イベントの処理を終了する
5.3 列数が一致する場合
次のステップ（レコードの主キールールチェック）に進む
6. レコードの主キールールチェック
6.1 対象テーブルのデータベースから全レコードの主キー情報を取得する
6.2 Excel ファイルの各行について、以下のルールをチェックする
処理区分が「追加」の場合：主キーがデータベースに既に存在するとエラー
処理区分が「更新」または「削除」の場合：主キーがデータベースに存在しないとエラー
6.3 エラーが発生した行は除外し、有効な行のみを保持する
7. データ形式チェック
7.1 対象テーブルの定義に基づき、Excel ファイルの各項目についてデータ形式（必須入力、データ型、最大桁数等）をチェックする
7.2 全てのチェックエラーを収集し、エラー情報として保存する
8. チェック結果の処理
8.1 全てのチェックでエラーが 1 件も発生しなかった場合
「ファイルのチェックが完了しました。補正データの登録ボタンを押下してアップロードを実行してください」の成功メッセージを表示する 有効なデータを保存し、エラー表示領域を非表示にする 8.2 1 件以上のエラーが発生した場合 「ファイルに誤りがあります。詳細は以下のエラーリストをご確認ください」のエラーメッセージを表示する エラーリストのうち、先頭 10 件を画面に表示する エラー件数が 10 件を超える場合、「合計 XX 件のエラーがあります。チェック結果ダウンロードボタンを押下して全件確認してください」のメッセージを追加表示する エラー表示領域を表示する 9. メッセージの表示 上記ステップで発生した全ての成功メッセージとエラーメッセージを、画面のメッセージ領域に表示する。
